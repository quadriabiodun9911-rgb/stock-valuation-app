## ✅ UX Release Sign-off Checklist (Mobile + Web)

Use this before shipping any release candidate.

### Core user journey

- [ ] User can complete: `Search stock → Analyze → Set alert` without confusion
- [ ] No critical screen has a dead-end state without a clear CTA
- [ ] First-time user can understand next action in under 3 seconds on Home/Analysis/Stock Detail

### Interaction and visual consistency

- [ ] Primary actions use touch targets with `minHeight >= 44`
- [ ] Secondary actions use touch targets with `minHeight >= 40`
- [ ] Disabled states are visually clear and tied to explicit validation rules
- [ ] Loading/error/success messaging follows consistent wording patterns

### Resilience and recovery

- [ ] All data screens support retry or pull-to-refresh
- [ ] Inline error banners appear for recoverable failures
- [ ] Empty states include a direct next-step action

### Platform readiness

- [ ] Backend health check passes: `GET /health`
- [ ] Expo mobile dev server starts cleanly
- [ ] Expo web startup (`npx expo start --web`) binds reliably to web endpoint
- [ ] No blocking CRC or bundling errors in startup logs

### Final validation

- [ ] Automated tests pass
- [ ] Manual smoke test passes on at least one iOS/Android target and web
- [ ] Release notes include known limitations and rollback command

## 📱 STEP 1: Backend Deployment (Choose One)

### **Option A: Render.com (Easiest) ⭐ Recommended**

#### 1.1 Create Render Account

```bash
# Go to https://render.com
# Sign up with GitHub account
# Click "New +" → "Web Service"
```

#### 1.2 Connect GitHub Repository

```
1. Click "Connect repository"
2. Select your GitHub repo (stock-valuation-app)
3. If not connected, authorize Render
4. Select repository: stock-valuation-app
```

#### 1.3 Configure Service

```
Name: stock-valuation-api
Environment: Python 3.11
Build Command: pip install -r backend/requirements.txt
Start Command: cd backend && python main.py
```

#### 1.4 Set Environment Variables

In Render dashboard, go to Environment:

```
ENVIRONMENT=production
DATABASE_URL=postgresql://user:pass@db.host/dbname
YFINANCE_CACHE_TTL=3600
CORS_ORIGINS=https://your-domain.com,exp://
```

#### 1.5 Deploy

```bash
# Push to GitHub
git push origin main

# Render auto-deploys on push
# Check deployment logs in Render dashboard
```

**Your API will be at:** `https://stock-valuation-api.onrender.com`

## 🗄️ STEP 2: Database Setup (Recommended)

### **PostgreSQL with Railway**

#### 2.1 Add Database to Railway Project

```
1. Go to Railway dashboard
2. Click "New" → "Database" → "PostgreSQL"
3. Railway auto-generates DATABASE_URL
4. Copy to backend environment variables
```

#### 2.2 Update Backend Code

```python
# backend/main.py
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

DATABASE_URL = os.getenv("DATABASE_URL", "sqlite:///./test.db")
engine = create_engine(DATABASE_URL)
SessionLocal = sessionmaker(bind=engine)
```

#### 2.3 Initialize Database

```bash
cd backend
python -c "from database import Base, engine; Base.metadata.create_all(engine)"
```

## 🔐 STEP 8: Security Best Practices

### 8.1 Environment Variables

```bash
# ✅ DO: Store in platform secrets
DATABASE_URL=postgresql://...
API_KEY=secret_key_123
CORS_ORIGINS=https://yourdomain.com

# ❌ DON'T: Commit to GitHub
git add .gitignore  # Exclude .env
```

### 8.2 API Authentication (Future)

```python
# Add JWT tokens for user accounts
from fastapi.security import HTTPBearer

security = HTTPBearer()

@app.get("/api/protected")
async def protected_route(credentials: HTTPAuthCredentials = Depends(security)):
    verify_token(credentials.credentials)
    return {"data": "protected"}
```

### 8.3 CORS Configuration

```python
# backend/main.py
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "https://yourdomain.com",
        "exp://",  # Expo
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 8.4 Rate Limiting

```bash
pip install slowapi

# Add to backend
from slowapi import Limiter

limiter = Limiter(key_func=get_remote_address)
app.state.limiter = limiter

@app.get("/smart-strategy")
@limiter.limit("10/minute")
async def smart_strategy():
    return smart_strategy_impl()
```

## 🚀 STEP 9: CI/CD Pipeline (GitHub Actions)

### 9.1 Create Workflow File

```bash
# Create .github/workflows/deploy.yml
mkdir -p .github/workflows
```

### 9.2 Auto-Deploy on Push

```yaml
# .github/workflows/deploy.yml
name: Deploy to Production

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Deploy Backend
        run: |
          git push https://api_token@render.com/deploy-hook
      
      - name: Deploy Mobile
        run: |
          cd mobile && npm run build
```

### 9.3 Enable Auto-Deploy

```
1. Platform (Render/Railway) automatically deploys on GitHub push
2. No manual action needed
3. Just push to main branch
```

## 🐛 Troubleshooting

### Backend Won't Start

```bash
# Check logs
render logs  # or railway logs, fly logs

# Common issues:
1. Missing dependencies: pip install -r requirements.txt
2. Database not connected: Check DATABASE_URL env var
3. Port binding: Platform assigns port automatically
```

### Mobile App Can't Connect

```bash
# Check API URL
echo $API_BASE_URL

# Verify backend is running
curl https://your-api.onrender.com/smart-strategy

# Check CORS settings in backend
# Ensure mobile URL is in CORS_ORIGINS
```

### Build Fails

```bash
# Clear cache and rebuild
cd mobile && rm -rf node_modules && npm install

# Check TypeScript errors
npx tsc --noEmit

# Test locally first
npx expo start
```
